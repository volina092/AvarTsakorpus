Recherche simple

Ce corpus est un ensemble de textes avec une annotation spéciale permettant de déterminer dans quels contextes, quels types de textes et à quelle fréquence un mot ou une construction spécifique est employé(e). Pour ce faire, il suffit d'entrer une requête de recherche dans les champs du panneau de recherche. Cette section explique comment effectuer des requêtes simples, c'est-à-dire des requêtes permettant de trouver dans le corpus un mot ou une classe de mots spécifique.

Le formulaire de requête pour la recherche d'un mot est composé de champs dans lesquels l'utilisateur peut saisir du texte. Les champs suivants sont disponibles :

  • Mot. Dans ce champ, vous pouvez saisir une forme de mot spécifique, par exemple, «идёт» (« il (elle) va » en russe). Dans ce cas, seule cette forme sera trouvée et non les formes «идти» (« aller ») или «шёл» (« il allait »). Il existe plusieurs modes de recherche de mots. Le premier consiste simplement à entrer un mot spécifique. Le deuxième permet d'utiliser le symbole *, qui signifie « un nombre quelconque de caractères quels qu'ils soient ». Par exemple, la requête «корпус*» (« corpus* ») trouvera tous les mots commençant par «корпус» : «корпус», «корпуса», «корпусный», etc. La troisième option consiste à utiliser des expressions régulières (obéissant à la syntaxe de REGEX Apache Lucene). Enfin, dans n'importe quelle requête, le signe ~ peut être utilisé comme négation, par exemple, «~корпус» rouvera tous les mots sauf «корпус».
  • Lemme. Le lemme est la forme de dictionnaire (canonique) d'un mot, par exemple, le lemme pour «идёт» (« il (elle) va ») sera «идти» (« aller »). Si vous entrez un lemme dans ce champ, vous trouverez toutes ses formes. Naturellement, une telle recherche ne fonctionnera que si le corpus a été lemmatisé. Les règles pour les requêtes utilisant le champ « Lemme » sont les mêmes que celles pour le champ « Mot ».
  • Grammaire{% if gloss_search_enabled %} et gloses{% endif %}: voir l'onglet Grammaire.
  • Autres champs. En fonction de l'annotation disponible dans le corpus, d'autres champs peuvent être présents dans le formulaire de requête. Tous les champs supplémentaires peuvent être consultés en cliquant sur le bouton . Si le paramètre peut prendre des valeurs arbitraires, le champ correspondant fonctionnera comme les champs « Mot » et « Lemme ». Si l'ensemble de valeurs est limité à une liste fermée, elles peuvent être sélectionnées dans le tableau en cliquant sur l'icône à la fin du champ — comme dans le champ « Grammaire ».
  • Le champ Position dans l’énoncé permet de chercher des mots à des endroits spécifiques à l'intérieur d'un énoncé, par exemple, au début (position 1) ou à la position de Wackernagel (position 2). Pour compter les positions depuis la fin de l’énoncé, il est possible d'utiliser des nombres négatifs : par exemple, -1 signifie le dernier mot de l’énoncé.

Si plusieurs champs sont remplis, tous les mots correspondant à tous les champs seront trouvés.

Options de recherche et d'affichage

La section de la barre latérale gauche permet de configurer les options de recherche et d'affichage. Les options pour la recherche simple comprennent le nombre de résultats par page (jusqu'à 100), le type de tri, la sélection de la translittération (dans les corpus qui permettent plusieurs variantes de translittération) et le type de présentation (normale ou glosée).

Recherche d’énoncés et de mots

Il existe deux types de recherches différents dans le corpus : la recherche d’énoncés et la recherche de mots (ainsi que de lemmes). Dans le premier cas, seront affichés tous les énoncés dans lesquels la plateforme de recherche a trouvé au moins un mot ou une combinaison de mots demandée par l'utilisateur. En cliquant et recliquant sur la flèche verte à côté de l’énoncé, vous pouvez voir son contexte, c'est-à-dire les énoncés voisins dans le texte (dans la plupart des corpus, le contexte sur un résultat peut être étendu un nombre limité de fois pour des raisons de protection des droits d'auteur). En décochant la case à gauche de l’énoncé celui-ci sera « désactivé ». Pour télécharger tous les résultats consultés, vous pouvez utiliser les liens « Télécharger au format CSV » et « Télécharger au format XLSX ». Les énoncés « désactivés » ne seront pas inclus dans cette liste.

Le bouton « Recherche de mots » affichera une liste de mots correspondant à la requête, sans répétitions, ainsi que des informations sur leur fréquence dans l'ensemble du corpus ou dans un sous-corpus si vous l'avez sélectionné. Une telle recherche peut répondre à des questions telles que « Quels verbes sont le plus souvent utilisés au passé ? ». Chacun des mots trouvés peut être immédiatement recherché dans le corpus en cliquant sur le bouton dans la ligne correspondante du tableau. Le bouton montrera la répartition de ce mot dans différents types de textes. La recherche de lemmes fonctionne de la même manière, mais au lieu de mots, elle montre des listes de lemmes.

Alignement avec audio/vidéo

Si l’énoncé que vous avez trouvé est aligné avec un fichier audio ou vidéo, vous pouvez voir ou écouter le segment correspondant en cliquant sur le texte de l’énoncé à n'importe quel endroit. Si vous avez élargi le contexte, vous pouvez voir ou écouter le segment entier. Dans ce cas, le segment qui est actuellement en cours de lecture sera en gras dans le texte. Dans le cas d'un enregistrement de dialogue où les répliques des participants se chevauchent, elles apparaitront dans l'ordre de leur début dans la transcription.

Recherche négative

Pour trouver des énoncés qui, ne contiennent pas certains mots ou classes de mots, vous pouvez activer l'option « Recherche négative », en cliquant sur le bouton . Cette option ne fonctionne que pour la recherche d’énoncés.

Historique

Vous pouvez consulter l'historique de vos recherches en cliquant sur l'icône . Pour répéter une recherche, cliquez sur les flèches à la fin de la ligne correspondante.

Téléchargement des résultats

Les énoncés trouvés peuvent être téléchargés dans des formats tels que CSV ou XSLX en cliquant sur le bouton или en bas de la page. Attention ! Seuls les résultats consultés dans le navigateur seront téléchargés. Par exemple, si vous avez ouvert les pages 1, 2 et 8, seuls les énoncés de ces trois pages seront inclus dans le fichier. Il n'est pas possible de télécharger tous les résultats en même temps. Ce n'est pas un bug, mais une décision délibérée : essayer de télécharger trop de résultats à la fois peut faire planter le serveur pendant longtemps.

Étiquettes grammaticales

Grammaire

Dans un corpus avec une analyse morphologique, chaque mot est associé à une série d'étiquettes — qui sont des abréviations qui indiquent quelles valeurs grammaticales (ou autres) sont exprimées. Par exemple, dans un corpus en russe, la forme verbale «идёт» peut avoir les étiquettes V (verbe), prs (temps présent), 3 (troisième personne) и sg (singulier). Les catégories qui seront étiquetées dans le corpus et les étiquettes correspondantes sont déterminées par les développeurs du corpus. Les utilisateurs familiers avec l'ensemble d'étiquettes utilisé dans un corpus spécifique peuvent manuellement entrer ces étiquettes dans le champ Grammaire, en les combinant avec des fonctions logiques telles que & ou , (virgule) pour la conjonction (et), | pour la disjonction (ou), et ~ pour la négation. Les parties de la requête peuvent être groupées à l'aide de parenthèses. Par exemple, une requête « (nom|gen),~sg » signifie « le mot doit être au nominatif ou au génitif, mais pas au singulier », s'il y a des étiquettes correspondantes dans le corpus. Les utilisateurs qui ne sont pas familiers avec l'ensemble d'étiquettes ou qui préfèrent une interface graphique peuvent ouvrir une fenêtre de sélection d'étiquettes à partir d'une liste, en cliquant sur un petit icône à la fin du champ de texte. Dans cette fenêtre, les étiquettes sont regroupées par catégories. Si vous sélectionnez plusieurs étiquettes dans une même catégorie, le corpus interprétera cela comme une disjonction (ou) ; les étiquettes de catégories différentes seront combinées par une conjonction (et). La requête correspondante sous forme de texte sera affichée dans le champ bleu en haut.

Gloses

Dans un corpus glosé, il est possible de rechercher des informations à partir des gloses. Cette recherche ressemble beaucoup à une recherche par des tags grammaticaux et recouvre en grande partie les mêmes aspects, mais présente quelques différences. Tout d'abord, lors d'une recherche à partir des gloses, il est possible de spécifier la disposition de plusieurs affixes ou de trouver des mots dans lesquels une glose apparaît plusieurs fois, alors que dans la recherche grammaticale, l'ensemble des tags est interprété comme un ensemble de valeurs non ordonnées sans répétitions. Deuxièmement, lors d'une recherche à partir des gloses, il est possible de spécifier non seulement la glose, mais aussi le morphème lui-même, ce qui peut être utile lorsque la même signification est exprimée par des allomorphes différents dans différents cas. Troisièmement, les développeurs du corpus peuvent prévoir des désignations généralisées pour les classes de gloses, par exemple, « CAS » pour tous les indicateurs de cas. Enfin, la correspondance entre les morphèmes/gloses et les valeurs grammaticales n'est pas toujours univoque, car plusieurs valeurs peuvent être cumulées dans un même morphème, et une seule valeur peut être exprimée simultanément par plusieurs morphèmes ou même ne pas être exprimée du tout.

Les utilisateurs familiers avec le système de glosage utilisé dans le corpus peuvent taper des requêtes de recherche dans le champ « Gloses ». Plusieurs gloses séparées par des tirets signifient que ces gloses doivent être trouvées dans le mot exactement comme elles ont été entrées. Par exemple, l'expression « GEN-PL » signifie « trouver tous les mots dans lesquels la glose PL suit immédiatement la glose GEN, mais pas l'inverse ». Entre accolades après la glose, il est possible de spécifier un morphème particulier : avec l'expression « GEN-PL{zzz} » seuls les mots dans lesquels la glose PL suit GEN et où le morphème glosé comme PL ressemble à zzz seront trouvés. Les expressions régulières peuvent être utilisées à l'intérieur des accolades. Au lieu d'une seule glose, il est possible de spécifier un ensemble de gloses en utilisant des parenthèses et l'opérateur | (ou), par exemple, « (NOM|GEN)-PL ». En outre, il est possible d'utiliser le symbole * avec la valeur « n'importe quel nombre de morphèmes », + avec la valeur « n'importe quel nombre positif de morphèmes » et ? avec la valeur « un seul morphème ». Enfin, le signe #, placé au début ou à la fin de l'expression, signifie la frontière du mot. Plusieurs expressions similaires peuvent être combinées à l'aide de fonctions logiques et de parenthèses. Comme pour la grammaire, au lieu de saisir manuellement les données, il est possible d'utiliser la liste des gloses. Les rectangles avec les gloses dans cette liste doivent être déplacés dans la zone bleue en haut.

Comment trouver plusieurs mots dans un énoncé

Pour trouver une construction ou simplement plusieurs mots à l'intérieur d'un énoncé, vous pouvez augmenter le nombre de mots dans la requête en cliquant sur le bouton le nombre de fois souhaité. Par défaut, il est considéré que dans une requête avec plusieurs mots, ces mots sont recherchés à n'importe quelle distance les uns des autres à l'intérieur d'un seul énoncé.

Dans ce cas, la recherche par énoncé renverra tous les énoncés dans lesquels chacun des mots demandés a été rencontré au moins une fois. Dans un corpus parallèle, il est possible de définir des langues différentes pour différents mots. Une requête multilingue de ce type signifiera « trouver tous les segments parallèles alignés où les énoncés dans les langues spécifiées contiennent les mots correspondants ». Par exemple, dans un corpus parallèle anglais-russe aligné par énoncé, vous pourriez trouver tous les énoncés où le mot anglais « James » a été traduit en russe par «Иаков».

La recherche de mots fonctionne de la même manière, mais au lieu d’énoncés, elle renvoie une liste de mots qui ont été trouvés à la place du mot n°1 de la requête dans ces énoncés. Une telle recherche peut être utile, par exemple, si vous voulez savoir quels mots sont le plus souvent utilisés à gauche ou à droite de la préposition «к» (dans le deuxième cas, cette préposition doit être le mot n°2 dans la requête, mais la distance par rapport au mot n°1 doit être négative). Étant donné que la recherche de mots nécessite de parcourir et de traiter tous les énoncés qui correspondent à la requête, une telle recherche peut être lente. Si le temps imparti pour la requête est écoulé pendant la recherche, vous verrez le résultat du traitement d'un échantillon aléatoire d’ énoncés au lieu du résultat complet.

Distances

En appuyant sur le bouton , vous pouvez spécifier une distance minimale et maximale entre les mots dans la requête. Par exemple, vous pouvez spécifier que le mot n°2 doit être situé dans une plage de 1 à 2 mots du mot n°1. Des distances négatives sont autorisées, ce qui signifie que le mot en question doit être situé à gauche de celui auquel la distance est indiquée. Indépendamment des distances saisies, la recherche de plusieurs mots est toujours limitée à un seul énoncé ; il est impossible de rechercher dans une même requête des mots situés dans différents énoncés. Si votre requête est relativement simple, vous n'avez rien d'autre à faire. Cependant, dans le cas d'une requête très complexe, vous obtiendrez également des exemples qui ne correspondent que partiellement à celle-ci. Ces exemples seront « désactivés » (ils seront semi-transparents), mais ils seront pris en compte dans le nombre total d'occurrences. Une requête est considérée comme « très complexe » s'il n'y a aucun mot qui figure dans toutes les contraintes de distance. De plus, même dans le cas de requêtes simples, il est possible d'inclure délibérément dans les résultats des exemples contenant tous les mots nécessaires, mais qui ne respectent pas les contraintes de distance, sous forme désactivée. Pour cela, il suffit de décocher la case « Distances entre les mots exactement comme dans la requête » dans l'onglet des paramètres.

Dans tous les cas, il convient de se rappeler que les requêtes à plusieurs mots, en particulier avec des distances spécifiées, peuvent être lourdes, de sorte que leur exécution peut prendre beaucoup de temps.

Comment choisir un sous-corpus

Pour sélectionner un sous-corpus, vous pouvez limiter votre recherche à un sous-ensemble de textes dans lesquels vous souhaitez faire votre recherche. Pour ce faire, cliquez sur le bouton « Sélectionner un sous-corpus ». Dans la fenêtre qui s'ouvre, vous pouvez soit limiter le sous-corpus en remplissant les champs de recherche, soit sélectionner (ou désélectionner) des textes spécifiques dans la liste (ces deux méthodes peuvent être combinées). Si vous avez sélectionné un sous-corpus, toutes vos requêtes seront recherchées uniquement dans les textes du sous-corpus sélectionné.
